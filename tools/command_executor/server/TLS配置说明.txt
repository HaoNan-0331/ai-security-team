【TLS 加密的作用】
  - 保护通信数据不被窃听
  - 防止中间人攻击
  - 验证服务端身份
  - 生产环境强烈建议启用


【配置步骤】

  方法一：使用环境变量（推荐）

  Windows:
  set USE_TLS=true
  set TLS_CERT=./certs/server.crt
  set TLS_KEY=./certs/server.key

  Linux/Mac:
  export USE_TLS=true
  export TLS_CERT=./certs/server.crt
  export TLS_KEY=./certs/server.key


  方法二：直接修改配置文件

  编辑 server/config.py，修改以下配置：

  USE_TLS = True                     # 启用TLS
  TLS_CERT = "./certs/server.crt"    # 证书文件路径
  TLS_KEY = "./certs/server.key"     # 私钥文件路径


【证书生成】

  自签名证书（适合内网测试）:

  1. 安装 OpenSSL
     Windows: https://slproweb.com/products/Win32OpenSSL.html
     Linux: sudo apt install openssl

  2. 运行生成脚本
     cd E:\knowlegdge_base\python_project\command_executor
     cd server
     generate_cert.bat

  3. 证书将生成在 certs/ 目录下


  手动生成证书（OpenSSL命令）:

  openssl req -x509 -newkey rsa:4096 -keyout certs/server.key -out certs/server.crt -days 3650 -nodes -subj "/CN=192.168.10.249"

  参数说明:
  -x509: 生成自签名证书
  -newkey rsa:4096: 使用4096位RSA密钥
  -days 3650: 有效期10年
  -nodes: 不加密私钥
  -subj: 证书主题，CN修改为你的服务端IP


【正式证书（公网部署）】

  如果服务端部署在公网，建议使用正式证书：

  1. Let's Encrypt (免费)
     https://letsencrypt.org/

  2. 阿里云/腾讯云证书
     购买域名后可申请免费DV证书

  3. 自建CA签名
     使用企业内部CA签发证书


【客户端配置】

  启用TLS后，客户端配置需要同步修改：

  {
    "server_host": "192.168.10.249",
    "server_port": 8765,
    "use_tls": true    ← 改为 true
  }


【验证TLS是否生效】

  1. 启动服务端，查看日志：
     [TLS] 已加载证书 - ./certs/server.crt
     WebSocket服务启动在 0.0.0.0:8765 - 协议: WSS (TLS加密)

  2. 启动客户端，查看日志：
     正在连接到服务端...
     使用TLS: True
     [连接] 正在连接服务端 - URL: wss://192.168.10.249:8765


【常见问题】

  Q: 启动时报错 "证书文件不存在"？
  A: 检查 certs/ 目录是否存在证书文件

  Q: 客户端连接失败？
  A: 确认客户端配置 use_tls 已设置为 true

  Q: 浏览器提示证书不安全？
  A: 自签名证书不受浏览器信任，需要手动信任。
     内网使用可以忽略此警告。

  Q: 如何查看证书信息？
  A: openssl x509 -in certs/server.crt -text -noout


【文件结构】

  server/
  ├── config.py              # 配置文件
  ├── websocket_server.py    # WebSocket服务
  ├── generate_cert.bat      # 证书生成脚本
  └── certs/                 # 证书目录
      ├── server.crt         # 证书文件
      └── server.key         # 私钥文件


【安全建议】

  1. 私钥文件权限保护
     chmod 600 server.key

  2. 证书定期更新
     建议有效期设置为1-3年

  3. 使用强加密算法
     RSA至少2048位，推荐4096位

  4. 生产环境使用正式证书
     不要使用自签名证书


【禁用TLS】

  如果不需要加密，设置：
  USE_TLS = false

  客户端同步修改：
  use_tls = false
