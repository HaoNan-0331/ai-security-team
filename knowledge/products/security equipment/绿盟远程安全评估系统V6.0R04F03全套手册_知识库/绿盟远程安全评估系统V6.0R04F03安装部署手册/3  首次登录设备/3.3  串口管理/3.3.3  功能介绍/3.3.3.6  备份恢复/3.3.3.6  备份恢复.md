系统诊断与自动恢复
系统诊断，即对系统出现的异常进行诊断与修复。完整的系统诊断过程需要较长时间，请提
前备份系统证书和重要的扫描数据，并耐心等待。
在控制台管理的主菜单中，进入 备份恢复 > 系统诊断，如图3-18所示。
图3-18 控制台管理 – 系统诊断与自动恢复
在系统诊断菜单中，管理员可以进行硬盘坏道检测和自动修复操作。
硬盘坏道的检测和自动修复需要较长时间。检测期间，所有服务将被停止，检测结束后重启
系统。
生成系统还原点
还原点在恢复RSAS的数据时使用。还原点的信息包括系统配置信息、资产信息、扫描模板
信息、扫描任务和用户信息等。
在控制台管理的主菜单中，进入备份恢复 > 生成还原点，如图3-19所示。管理员可以手工
创建用户还原点。

重新安装系统
仅硬件型号支持此功能。重新安装系统，即将整个系统初始化为出厂配置，仅保留网络配置、
证书文件和已使用的IP地址个数，其他数据会全部丢失。在RSAS出现故障时，可以使用此
功能。

3.3.3.7 退出系统
控制台的各项参数配置完毕，返回到主菜单，将光标移到 退出，按回车键即可退出控制台
管理界面，如需再次配置，请重新登录。
3.4 导入证书-首次
在正常使用系统之前，需要导入正确的证书。
步骤 1 打开浏览器，使用HTTPS方式访问管理口的IP地址（例https://192.168.1.1）。
步骤 2 选择接受正常风险后，进入证书导入页面。
步骤 3 导入与HASH匹配的扩展名为.lic的证书文件即可。
----结束

3.4.2 认证方式-VM 型号

3.4.2.1 加密狗证书认证
加密狗证书认证的具体操作如下：
步骤 1 给虚拟化设备添加加密狗USB设备。
步骤 2 打开浏览器，使用HTTPS方式访问管理口的IP地址（例https://192.168.1.1）；接受正常风
险。
步骤 3 在“选择认证方式”对话框中，选择“证书认证”，如图3-23所示。
