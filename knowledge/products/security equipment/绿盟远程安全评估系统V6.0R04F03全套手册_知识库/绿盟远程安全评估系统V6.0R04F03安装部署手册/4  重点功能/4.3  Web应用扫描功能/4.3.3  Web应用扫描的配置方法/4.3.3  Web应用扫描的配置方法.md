4.3.2.1 发包速率范围
Web应用扫描的发包速率受网络环境的影响非常大，按照网络最差的情况即超时30秒计算，
假设一分钟可以发送两个包，则每分钟的发包速率为：2*设置的并发web任务数/分钟；在
网络情况极端好的情况下，假设2毫秒可以发送一个包，按照一分钟发送30000个包进行计
算，则每分钟的发包速率为：30000*设置的并发web任务数/分钟。
由此得出，web应用扫描的发包速率范围为【2*并发web任务数/分钟】~【30000*并发web
任务数/分钟】。
这个发包速率是以RSAS的出口来测算的，而不是从客户那边的接收端测算，客
户那边接收的发包量不排除会有中间设备的复制重发或者拦截等导致数量增加
或减少的情况。
4.3.2.2 不支持登录扫描的 Web 站点
不支持登录扫描的Web站点的4种情况：
⚫ 登录会跳转至统一认证身份平台的站点。
⚫ 插入硬件设备认证的站点，如插入key认证的站点。
⚫ 认证信息没有存储在请求或响应头中的站点。
⚫ RSAS系统版本V6.0R04F03以前不支持登录扫描前后端分离站点。
4.3.2.3 支持 web 弱口令猜测的站点类型
⚫ web口令猜测支持的类型：表单填充的、basic认证的、已知应用的（Apache APISIX 存
在弱口令、通达OA存在弱口令、检测到webmin存在弱口令等等插件里面可搜索到的
弱口令检测）。
⚫ 不支持口令猜测的类型：带验证码的站点。

4.3.3.3 通过自定义请求头进行登录扫描
某些站点登录时不仅仅使用cookie进行认证，还需要请求头中的其他信息，例如bearer认
证、token认证等。扫描这类站点时可以通过自定义请求头进行登录扫描，具体步骤如下：
步骤 1 手动在浏览器上登录目标站点，登录后按F12键打开开发工具，复制请求头中的全部信息，
如图4-50所示，其中，蓝色字体是key，黑色字体是value，一个请求头中包含许多字段。
图4-50 复制请求头
步骤 2 进入 新建任务 > Web应用扫描 > 高级选项，在自定义header中输入刚刚复制的请求头信
息，如图4-51所示。
图4-51 粘贴请求头
填写时，每一行代表请求头中的一个key和value，因此多行字段需要多次添加，
填写一行后单击右侧的“加号”图标进行添加。
