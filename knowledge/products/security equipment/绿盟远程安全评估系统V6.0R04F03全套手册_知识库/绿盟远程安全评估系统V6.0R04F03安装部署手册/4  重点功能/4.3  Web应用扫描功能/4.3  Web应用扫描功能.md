4.3 Web 应用扫描功能
RSAS的Web应用扫描功能为用户的互联网网站提供远程安全扫描和安全检查，能够根据站
点管理者的监管要求，对目标站点进行不间断的页面爬取、分析、匹配、检测漏洞，为构建
安全的网站提供完善的监测方案。

4.3.1 扫描流程简介
web应用扫描任务流程如图4-39所示。
图4-39 Web应用扫描任务流程
web应用扫描任务具体执行步骤如下：
步骤 1 进行站点存活性判断：存活性判断主要是为了迅速给出站点是否可达的结论，不可达则不会
再往下探测，避免做一些无用功。
步骤 2 页面爬取时，主动识别页面类型（图片，外部链接，二进制文件，其它纯静态文件等）、尝
试猜测一些无法从其他页面解析出来的但可能存在的目录并做好标记。
步骤 3 探测点发现：不同的插件有针对性地在请求中寻找不同的探测点，可能的探测点有：URL路
径，GET方法URL中的参数，POST方法请求体中的参数，请求头中的字段，cookie中的键
值，响应体等等。
步骤 4 漏洞检测：每个具体的漏洞都有相应的一个插件来进行具体的检测。插件根据得到的探测点，
有针对性地构造特殊的网络请求，使用远程网站漏洞扫描检测技术进行漏洞检测，判断是否
存在相应的漏洞。
步骤 5 扫描结束，将漏洞结果写入到数据库，并以报表的形式展现出来。
----结束
4.3.2 注意事项
因为web扫描会模拟人用浏览器访问站点时进行的操作去请求网站链接，所以会在目标网
站上存有一些行为数据，尤其是在网站存在漏洞或者执行登录扫描时，例如：当使用管理员
权限的账号进行登录扫描时，该账号可以单击按钮发送测试邮件，那么RSAS就有权限去请
求这个按钮源码中的链接，就会触发发送测试邮件的操作。
因此，在执行web扫描时应注意以下几点：
⚫ 尽量使用备用或测试的站点环境进行web扫描。
⚫ 扫描前备份站点数据。
⚫ 不需要登录的站点就不用登录扫描。
⚫ 必须要登录才能访问站点页面的站点使用低权限用户登录。
⚫ 手动开启表单填充的话会在目标站点的表单中填充数据（默认未启用）。

4.3.2.1 发包速率范围
Web应用扫描的发包速率受网络环境的影响非常大，按照网络最差的情况即超时30秒计算，
假设一分钟可以发送两个包，则每分钟的发包速率为：2*设置的并发web任务数/分钟；在
网络情况极端好的情况下，假设2毫秒可以发送一个包，按照一分钟发送30000个包进行计
算，则每分钟的发包速率为：30000*设置的并发web任务数/分钟。
由此得出，web应用扫描的发包速率范围为【2*并发web任务数/分钟】~【30000*并发web
任务数/分钟】。
这个发包速率是以RSAS的出口来测算的，而不是从客户那边的接收端测算，客
户那边接收的发包量不排除会有中间设备的复制重发或者拦截等导致数量增加
或减少的情况。
4.3.2.2 不支持登录扫描的 Web 站点
不支持登录扫描的Web站点的4种情况：
⚫ 登录会跳转至统一认证身份平台的站点。
⚫ 插入硬件设备认证的站点，如插入key认证的站点。
⚫ 认证信息没有存储在请求或响应头中的站点。
⚫ RSAS系统版本V6.0R04F03以前不支持登录扫描前后端分离站点。
4.3.2.3 支持 web 弱口令猜测的站点类型
⚫ web口令猜测支持的类型：表单填充的、basic认证的、已知应用的（Apache APISIX 存
在弱口令、通达OA存在弱口令、检测到webmin存在弱口令等等插件里面可搜索到的
弱口令检测）。
⚫ 不支持口令猜测的类型：带验证码的站点。
