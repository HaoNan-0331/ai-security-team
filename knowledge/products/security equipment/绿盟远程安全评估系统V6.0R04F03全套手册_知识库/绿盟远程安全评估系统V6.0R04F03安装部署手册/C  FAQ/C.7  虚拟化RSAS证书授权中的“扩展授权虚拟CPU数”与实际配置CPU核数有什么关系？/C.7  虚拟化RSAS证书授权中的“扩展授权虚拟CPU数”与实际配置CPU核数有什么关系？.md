C.5 任务列表中任务数达最大值后，是否还能新建扫描任务？
A：不能，需要手动删除历史任务，释放任务数后才能重新下发新任务。
C.6 证书中的“扩展授权扫描口数”与网口数有什么关系？
A：RSAS实际可用的扫描口=授权扫描口数+1。
C.7 虚拟化 RSAS 证书授权中的“扩展授权虚拟 CPU 数”与实际配
置 CPU 核数有什么关系？
A：RSAS实际可用的CPU核数=扩展授权虚拟CPU数+2。

C.9 怀疑扫描结果漏报、误报时需要收集哪些信息？
当怀疑扫描结果出现误报和误漏报问题时，需要绿盟科技售后协助排查，可以通过调试及抓
包获取测试过程中详细的插件、引擎等信息进行有效的排查。
抓包
RSAS支持从设备接口直接抓包，用于分析、调试问题。使用抓包功能前，请先在 系统管理 >
配置 > 系统配置 页面开启“专业参数”，开启后抓包功能页面才会显示。
进入 系统管理 > 服务 > 系统服务 页面，在“网络抓包”区域，进行网络抓包。
⚫ 单击【开始抓包】按钮，RSAS设备开始进行数据抓包。
RSAS设备将对所有通过处于启用状态的网络接口的数据进行抓包。
⚫ 单击【结束抓包】按钮，结束抓包。
系统默认一次最多抓取100万个数据包，达到最大数据包前未手动停止抓包，RSAS设
备将强制停止抓包。
