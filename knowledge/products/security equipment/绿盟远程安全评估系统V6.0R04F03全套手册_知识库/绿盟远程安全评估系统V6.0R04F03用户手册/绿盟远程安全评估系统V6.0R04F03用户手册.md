1.1 产品特点
RSAS作为新一代的漏洞扫描和配置管理的产品，能够全面发现网络中存在的各种脆弱
性问题，快速定位网络风险。
全面脆弱性发现
RSAS能够全方位检测IT系统存在的脆弱性，发现扫描目标中存在的安全漏洞、安全配
置问题、Web应用安全漏洞、存在的弱口令、代码安全缺陷，收集系统不必要开放的帐
号、服务、端口。
直观脆弱性展示
RSAS通过绿盟科技专利安全风险计算方法，对网络系统中多个方面的安全脆弱性统一
进行分析和风险评估，给出总体安全状态评价，全面掌握信息系统安全风险，形成整体
安全风险报告。帮助管理员先于攻击者发现安全问题，及时进行修复。
RSAS的脆弱性分析结果以仪表盘方式展示，从风险发生区域、类型、严重程度进行不
同维度的分类分析展示，用户可以全局掌握安全风险，关注重点区域、重点资产，优先
修复严重问题。当需要定位具体安全脆弱性时，可以单击仪表盘的风险数据，定位具体
IP地址的漏洞数据。

清晰的资产管理
RSAS使用IP地址对象的概念，通过风险视图来实现资产管理，以此作为系统和网络真
实状况之间的纽带。用户可以预先定义网络逻辑结构，然后在日常工作中使用资产视图
实现风险管理，或者通过评估任务自动发现生成的资产树，从而实现风险管理。
资产管理的应用体现在多个方面：
⚫ RSAS在执行扫描任务时可以自动发现网络中的IP地址信息，并将资产信息自动更
新至资产树中。
⚫ 管理员可以通过查看资产树来了解自己的网络资产状况。
⚫ 风险值计算方法和资产树中的资产重要性权值关联，准确计算风险等级。
⚫ 生成报表时，RSAS会根据IP地址找到相应的资产，读取并显示扫描目标的节点名
称和节点所属管理员。在发现漏洞时，管理员可以快速定位资产并找到相关负责人
迅速修补漏洞。
丰富的知识库
绿盟科技NSFOCUS安全小组，有多位专职的研究员进行漏洞跟踪和漏洞前瞻性研究，
到目前为止已经独立发现了40余个关于常见镜像、操作系统、数据库和网络设备的漏
洞，并且为国际上的知名网络安全厂商提供相关漏洞的规则支持。NSFOCUS安全小组
负责RSAS的漏洞知识库和检测规则的维护，除定期的每两周的升级外，重大漏洞的升
级在全球首次发现后两天内完成。
依靠专业的NSFOCUS安全小组的研究积累，RSAS产品知识库已经有超过18万条系统
漏洞信息，涵盖所有主流基础系统、应用系统、网络设备等网元对象；知识库中还提供
7大类、30多种产品、上百个版本的系统的配置检查库，提供绿盟科技作为专业安全厂
商的加固修复建议，以及多个行业的安全配置检查标准。
RSAS支持对主流代码文件进行审计，能够发现代码中存在的安全缺陷和不符合编码规
范的内容。
识别非标准端口
RSAS应用先进的非标准端口识别技术、以及丰富的协议指纹库，能够快速准确的识别
非标准端口上的应用服务类型，并进一步进行漏洞检测，极大的避免了扫描过程中的漏
报和误报。

一般来说，一个 URL 地址由一个五元组（page，method，query-name，

 page=http://www.nsfocus.com/test.php（page=协议+域名+路径文件）
 method=GET
 query-name=login
 query-value=admin
链接消重策略  post-data=NULL
那么对于消重等级来说：
 0：对page敏感
 1：对page，method敏感
 2：对page，method，query-name敏感
 3：对page，method，query-name和query-value敏感
 4：对page，method，query-name，query-value和post-data敏感
消重等级越高，则URL地址的相同因素就要越多，当设定等级为0时，只
要两个URL的page相同，就认定这两个URL是同一个URL，无需再考虑
后续的参数值。

− 如果设备不能访问SSH服务器，但是某台与设备路由可达的主机可以访问SSH
服务器，则在该主机中运行PortGo.exe并进行相关设置后，即可将设备的SSH
口映射到SSH服务器。

一个技术支持端主机可以同时连接多台映射设备，但是多个技术支持端主机不能
同时连接到一台映射设备中。

FAQ

B.1 系统管理
Q1：为何不能访问 Web 界面？
A：请检查以下几方面内容是否正确：
⚫ 是https连接，而不是http连接。
⚫ IP、网关、DNS参数是否配置正确。
⚫ 请检查网络连接是否正确。
Q2：通过串口线为何不能登录控制台？
A：请检查以下几方面内容是否正确：
⚫ com口是否选择正确。
⚫ 端口属性、各项参数值是否正确。
⚫ 更换一根新的串口线，重新尝试。
Q3：什么权限可以进行证书导出的操作？
A：只要有证书管理权限，就可以进行导入导出操作。
B.2 任务扫描
Q1：下达评估任务时是否会限制扫描目标范围？
A：不会。所有型号的RSAS设备在下达任务时，其目标范围都可以下达若干个 B类地
址，但不能下达A类地址。而实际能得到扫描结果的IP个数是由证书来控制的。那些
没有得到扫描结果的IP地址会通过在线报表详细列出来。

Q2：什么是扫描任务？针对他的操作有哪些？
A：扫描任务指的是一次扫描的配置与执行结果，包括被扫描的主机范围、被扫描的策
略、扫描的参数信息以及其执行的扫描结果等。针对任务的操作有任务删除、重新扫描、
复制任务、任务导出、任务导入、汇总查看。
Q3：为何网络中真实存在的主机，扫描结果却显示不存活？
A：检查是否为跨网段扫描，中间可能有防火墙、IPS等设备将RSAS发送的数据包丢
弃。建议修改网络设备配置，RSAS发送的数据包都设置为放行。
Q4：导致 RSAS 产品扫描速度慢的原因有哪些？
A：大致有以下几方面原因：
⚫ 有防火墙
⚫ 端口扫描范围
⚫ 扫描策略
⚫ 最大主机并发数
Q5：“端口扫描速度”功能解释？
A：“端口扫描速度”这个功能点存在的原因是：Netscreen防火墙会认为RSAS的扫描
活动是syn flood或者connect flood，在进行端口扫描阶段就把RSAS的IP地址自动封
闭，所以提供了“慢速扫描”来穿透Netscreen防火墙。
Q6：为什么端口扫描策略中选中的 UDP 端口没有被扫描？
A：前提是必须在评估任务的端口扫描配置中选中“进行UDP扫描”。
Q7：暂停后继续扫描该任务的起始时间如何计算？
A：暂停后，继续扫描该任务的起始时间仍为任务最初开始执行的时间。
Q8：在任务列表中任务号没有连续递增？
A：因为在执行数据库修复操作中，在检测数据库的时候系统后台执行了部分数据库检
查操作。这个情况不会影响产品功能的使用。

C
安全等级保护
RSAS设备的信息系统安全等级保护功能支持国家等保标准的前四级。
不同安全保护等级的信息系统，其对业务信息的安全性要求和系统服务的连续性要求是
有差异的；即使相同安全保护等级的信息系统，其对业务信息的安全性要求和系统服务
的连续性要求也有差异。依据国家等级保护标准，第一到第四等级的信息系统应具备的
基本安全保护能力如下：
⚫ 第一级安全保护能力
应能够防护系统免受来自个人的、拥有很少资源的威胁源发起的恶意攻击、一般的
自然灾难、以及其他相当危害程度的威胁所造成的关键资源损害，在系统遭到损害
后，能够恢复部分功能。
⚫ 第二级安全保护能力
应能够防护系统免受来自外部小型组织的、拥有少量资源的威胁源发起的恶意攻
击、一般的自然灾难、以及其他相当危害程度的威胁所造成的重要资源损害，能够
发现重要的安全漏洞和安全事件，在系统遭到损害后，能够在一段时间内恢复部分
功能。
⚫ 第三级安全保护能力
应能够在统一安全策略下防护系统免受来自外部有组织的团体、拥有较为丰富资源
的威胁源发起的恶意攻击、较为严重的自然灾难、以及其他相当危害程度的威胁所
造成的主要资源损害，能够发现安全漏洞和安全事件，在系统遭到损害后，能够较
快恢复绝大部分功能。
⚫ 第四级安全保护能力
应能够在统一安全策略下防护系统免受来自国家级别的、敌对组织的、拥有丰富资
源的威胁源发起的恶意攻击、严重的自然灾难、以及其他相当危害程度的威胁所造
成的资源损害，能够发现安全漏洞和安全事件，在系统遭到损害后，能够迅速恢复
所有功能。
基本安全要求是针对不同安全保护等级信息系统应该具有的基本安全保护能力提出的
安全要求，根据实现方式的不同，基本安全要求分为基本技术要求和基本管理要求两大
类。其中基本技术要求根据保护侧重点的不同细分为三类：
⚫ 保护数据在存储、传输、处理过程中不被泄漏、破坏和免受未授权的修改的信息安
全类要求（简记为S）。
⚫ 保护系统连续正常的运行，免受对系统的未授权修改、破坏而导致系统不可用的服
务保证类要求（简记为A）。
