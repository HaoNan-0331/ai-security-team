# 证书

公钥基础设施（Public Key Infrastructure,简称PKI）技术采用证书管理公钥，通过第三方可信任机构--认证中心CA(Certificate Authority)，把用户的公钥和用户的其他标识信息（如名称、E-mail、身份证号等）捆绑在一起，在Internet网上验证用户的身份。目前通用办法是采用建立在PKI基础之上的数字证书，通过把要传输的数字信息进行加密和签名，保证信息传输的机密性、真实性、完整性和不可否认性，从而保证信息的安全传输。

设备上的PKI本地证书功能是：当设备作为PKI客户端时，选择本地证书作为设备的身份标识，并且验证从其他主机接收到的证书的合法性。这相当于IE浏览器中的证书项功能。主要包含三项配置：导入用户证书、导入第三方CA证书、导入第三方CA的证书撤销列表（Certificate Revocation List,简称CRL）。这三个功能是相对独立又相互联系，即可以根据具体需要，导入不同的本地证书、不同的CA证书、不同的CRL，但要验证某个终端证书时，需要导入该终端证书的CA证书、CRL，以便对该终端证书进行验证。

在系统菜单点击“对象>证书>本地证书”，进入本地证书页面，显示设备上已有的本地证书信息，在该页面可以实现对本地证书的导入、导出、查看和删除。

在本地证书页面点击<导入>，可以选择三种格式的证书上传。

PKCS12格式证书上传。

本地证书配置项及详细说明如下：

在本地证书页面点击，查看某一个证书的具体信息，显示证书详细信息。

在本地证书页面点击，可以导出证书。

在本地证书页面选中要删除的证书文件，点击<删除>或者点击证书后的，在弹出界面点击确定可以实现证书删除，如果证书前复选框为灰色或者不存在，表明证书正在被引用，无法删除。

在系统菜单点击“对象>证书>本地CA中心”进入本地CA中心页面，显示设备上已有的本地CA证书信息，在该页面可以实现对本地证书的导入、导出、查看和删除。

在本地CA中心页面点击<导入>，导入CA中心证书，可以选择两种上传CA证书的方式。

本地CA证书配置项及详细说明如下：

在本地CA中心页面点击查看某一个证书的具体信息，显示证书详细信息。

在本地CA中心页面点击，可以导出证书。

在本地CA中心页面选中要删除的证书文件，点击<删除>或者点击证书后的，在弹出界面点击确定可以实现证书删除，如果证书前复选框为灰色或者不存在，表明证书正在被引用，无法删除。

虚拟化防火墙支持导入SSL证书，验证SSL证书的有效性，防止欺骗攻击。导入的证书支持PKCS12格式和证书密钥分离两种。

系统菜单点击“对象>证书>SSL解密证书>转发根CA”，进入转发根CA页签，显示设备上已导入根CA证书，在转发根CA页面可以实现对证书的导入、导出、查看和删除。

在转发根CA页面点击<导入>，导入根CA证书，类型格式支持PKCS12和证书密钥分离。

转发根CA配置项及详细说明如下：

在转发根CA页面点击查看某一个证书的具体信息，显示证书详细信息。

在转发根CA页面点击，可以导出证书。

在转发根CA页面选中要删除的证书文件，点击<删除>或者点击证书后的，在弹出界面点击确定可以实现证书删除，如果证书前复选框为灰色或者不存在，表明证书正在被引用，无法删除。

系统菜单点击“对象>证书>SSL解密证书>受信任根CA”，进入受信任根CA页签，显示设备上内置的受信任根CA颁发的证书。在受信任根CA页面可以实现对证书的导入、导出、查看和删除。

受信任根CA信息项及详细说明如下：

在受信任根CA页面点击查看某一个证书的具体信息，显示证书详细信息。

在受信任根CA页面点击，可以导出证书。

在受信任根CA页面选中要删除的证书文件，点击<删除>或者点击证书后的，在弹出界面点击确定可以实现证书删除。如果证书前复选框为灰色或者不存在，表明证书正在被引用，无法删除。

证书撤销列表（Certificate Revocation  List，简称CRL），由于密钥被泄露、业务终止等原因，CA可通过撤销证书立即终止证书的使用，在此情况下CA需要公布CRL来声明该证书是无效的，并列出不能再使用的证书。

在系统菜单点击“对象>证书>导入CRL”，进入导入CRL页面，显示设备上已有的CRL信息，在该页面可以实现对CRL的导入、导出、查看和删除。

在导入CRL页面点击<导入>，导入CRL。

在导入CRL页面点击查看某一个CRL证书的具体信息，显示证书详细信息。

在导入CRL页面点击，可以导出证书。

在导入CRL页面选中要删除的证书文件，点击<删除>或者点击证书后的，在弹出界面点击确定可以实现证书删除，如果证书前复选框为灰色或者不存在，表明证书正在被引用，无法删除。
