# 入门指南

用户可通过运行Internet浏览器的任何计算机使用HTTP或一个安全的HTTPS连接，便能够配置并管理虚拟化防火墙（VNGFW）。在进行Web管理前，必须配置虚拟化防火墙设备使其能够接受来自指定接口的HTTP或HTTPS管理。

推荐使用Mozilla22.0及以上版本、Chrome27.0浏览器，最佳显示分辨率为1920×1080。

出厂的设备有默认的配置。这些默认配置保证用户不需要进行额外配置就能够通过CLI或Web对虚拟化防火墙设备进行管理、配置。CLI包括Console、SSH。Web支持HTTPS协议管理。

系统默认的管理员用户为admin，密码为admin。用户可以使用这个管理员账号从任何地址登录设备，并且使用设备的所有功能。

系统默认的审计员用户为audit，密码为admin.audit。用户可以使用这个账号对日志系统进行审计。

系统默认的用户管理员用户为useradmin，密码为admin.user。用户可以使用这个账号配置系统管理员。

Web系统菜单由导航条、菜单栏、和主内容区页面组成，每个导航标签里有相应的一个或多个子项目。当点击一个导航页如监控，左侧显示菜单项目，右侧为主内容区。

导航提供虚拟化防火墙设备的信息统计和主要配置选项。

除“概况”之外，其他导航页面左侧都有可折叠的浮动“菜单栏”。菜单栏中可以选择不同的配置项目进行细节配置或查看。

主内容区是显示配置和信息的界面，在不同导航项目中有不同的表现形式。在“监控”导航标签下，监控信息主要以图表的形式直观的展现出来，不同项目信息以不同颜色加以显示，方便查看。

在“日志”、“策略”、“网络”等导航标签下，信息主要以表格的形式展现，更加贴近用户的使用习惯。

虚拟化防火墙在有AV病毒、IPS入侵、DDOS攻击、FLOOD攻击时通过警示，显示攻击次数，用户点击按钮，能够及时发现虚拟化防火墙遭受的攻击类型、数量及生成日期。

系统菜单中点击按钮，能够及时看到攻击日志告警。点击<更多>，查看更多的攻击类型。点击<清除所有提醒>，清除告警提示。点击对应的告警提示日志，可跳转相应的日志页面。

虚拟化防火墙通过设置向导快速配置设备接口Ge0/0地址及SD-WAN管理参数，实现快速上网和集中管理。接口配置支持三种上网方式，分别是PPPoE、DHCP和静态IP。

在系统菜单点击图标，进入设置向导配置页面。

在设置向导页面，选择接口Ge0/0获取地址的方式，当前配置完成后点击<下一步>进入下一项云平台参数配置。

配置云平台参数完成后，点击<下一步>，进入配置确定页面。

点击<确定>后，可以在“网络>接口>物理接口”查看第一步中下发的Ge0/0接口配置，在“系统>SD-WAN”中可以查看第二步中配置的云平台配置及通信状态。

提示：

设备获取地址的三种方式可参考物理接口。

系统菜单中点击按钮，点击<确定>按钮后，可以将设备当前的配置及时保存。

提示：

除手动保存配置，虚拟化防火墙还默认支持实时保存配置，详见系统管理设定。

系统菜单中点击按钮，再次点击<确定>按钮，当前已登录用户可注销登录，退回登录页。
