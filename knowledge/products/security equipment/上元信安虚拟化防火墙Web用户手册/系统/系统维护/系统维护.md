# 系统维护

系统维护是为防止系统在运行过程中发生错误或故障，提供一种监控和维护手段。系统维护包含配置文件、升级与重启、授权、诊断工具、抓包工具、信息反馈、信息监控、PING工具、Trace工具、Token管理功能。

支持配置文件的导入导出和备份配置功能。

在系统菜单点击“系统>系统维护>配置文件”，进入配置文件页面。

配置文件导入导出：点击<导出>按钮，可以对设备当前配置进行导出，导出内容为txt文本文件。

点击<选取文件>按钮，选择保存在PC本地的正确配置文件，点击<导入配置>按钮可将配置导入设备，导入后的配置文件在设备重启后生效。

备份配置：点击<备份>按钮即可对当前设备中配置进行备份，点击按钮可将配置下载到本地，点击按钮可将当前配置替换为备份的配置。

支持对设备版本进行升级、支持查看版本升级记录；同时支持特征库版本自动升级及手动升级；支持页面重启设备、恢复出厂设置。

固件升级中不仅支持设备版本升级，还支持双系统备份，且在系统切换中可实现配置的自动迁移。该场景是为保证设备的稳定性，在单个设备上进行双系统备份，以达到一个系统崩溃，备用系统可以切换运行的特性。

系统菜单点击“系统>系统维护>升级与重启>固件升级”，进入固件升级配置页签。

软件镜像用于升级设备版本，点击<选取文件>按钮来选择版本文件，点击<升级版本>开始版本升级操作，成功后需要手动重启设备，使新版本生效，可在下方升级历史中查看设备升级历史记录。

系统备份用于双系统备份，点击<备份当前系统>按钮进行系统备份，在启动选项子目录下，点击下次启动版本右侧下拉框，可以展示备份版本。

诊断工具主要是为方便管理员对业务流程进行追踪、调试使用。

在系统菜单点击“系统>系统维护>诊断工具>诊断工具”，进入诊断工具配置页签。“协议”选项有ANY、TCP、UDP、ICMP、OTHER，默认显示为ANY。

诊断工具配置项及详细说明如下：

完成诊断工具配置，并有DEBUG结果后，点击“系统>系统维护>诊断工具>诊断信息导出”，进入诊断信息导出页签，可以导出诊断信息。

点击<导出>按钮，可导出诊断信息到终端本地。

设备出现异常时，会在异常信息导出页面生成异常信息文件方便用户下载并交给专业人士查看问题原因。

在系统菜单点击“系统>系统维护>诊断工具>异常信息导出”，进入异常信息导出页签。

设备出现异常时会在异常信息导出页面生成数据，点击图标下载文件到本地，点击图标删除数据。

用户可以通过抓包工具指定过滤条件，抓取实际网络中的数据包，便于分析网络状态，追踪网络问题。

在系统菜单点击“系统>系统维护>抓包工具”，进入抓包工具配置页面。

抓包工具配置项及详细说明如下：

生成数据的右侧“操作”列，点击图标下载数据到本地，点击图标可删除已生成的数据。

提示：

物理接口中如果不选择添加接口，默认抓取所有接口的数据包。

抓包结果中最多支持抓10个包，每个包大小不超过10M。

在系统菜单中点击“系统>系统维护>信息反馈”，进入信息反馈配置页面。

信息反馈配置项及详细说明如下：

点击<应用>后，收件人可接收到信息反馈邮件。

提示：

配置信息反馈的前提：完成告警邮件配置，且告警邮箱可成功发送邮件。

信息监控主要监控CPU占用率、内存占用率、硬盘占用率、流量、连接数、报文大小，这些告警配置如果超过规定值会在本地日志、Syslog日志、E-mail报警中产生告警信息，方便用户监控设备。

在系统菜单点击“系统>系统维护>信息监控”，进入信息监控配置页面。

信息监控配置项及详细说明如下：

点击<应用>后设备实际值超过告警配置值后触发告警，点击“日志>系统日志”进入系统日志。页面可查看到告警事件。

提示：

在配置信息监控前，完成日志过滤配置。
