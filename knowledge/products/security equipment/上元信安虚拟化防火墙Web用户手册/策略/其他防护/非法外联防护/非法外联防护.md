# 非法外联防护

非法外联是指内网主机通过非法连接访问Internet或者通过合法连接访问非法Internet的行为，非法外联行为分为物理非法外联和逻辑非法外联2种，物理非法外联指内网主机在未授权的前提下，通过网络设备建立一条进出内网的通路，非法外联行为有很多种，如拨号上网、双网卡上网、GPRS、红外等；逻辑非法外联指内联网主机通过正常途径访问Internet的不良信息，包括反动、暴力网站和游戏、聊天等。非法外联和非法接入等行为，很容易对物理隔离和逻辑隔离的内部网络造成损害。

非法外联的危害：

机密信息泄漏用户通过非法外联的计算机，主动或被动地外发内部涉密资料，给组织造成重大损失；

引入安全问题，非法外联计算机的存在为病毒、木马攻击内网提供理想通道，病毒或木马可以借助终端用户违禁使用U盘、擅自拨号进行互联网访问、随意浏览网站、随意下载网站软件等手段，乘虚而入，攻入内网，严重威胁到内网的稳定运行和内部数据的安全。

虚拟化防火墙通过配置非法外联策略设定允许终端或者服务器外联的地址、外联地址的服务及端口来定义正常外联行为，保证内部网络的安全。

在系统菜单点击“策略>其他防护>非法外联防护>非法外联策略”，进入非法外联策略配置页签。显示设备上配置的所有非法外联策略，在该页面可以对非法外联策略进行新建、删除、编辑和启用/禁用。

在非法外联策略页面点击<新建>创建新的非法外联策略，或在右侧“操作”列点击编辑已有的非法外联策略。

开启允许外联规则状态后，可通过点击<新建>创建具体规则或点击右侧操作列点击编辑已有的规则。

也可以通过点击<删除>或右侧操作列的删除具体规则。

非法外联策略配置项及详细说明如下：

注意

配置IPv4主机和域名，IPv6主机地址类型，配置端口数量暂定128条，每条非法外联策略中可配置规则为：IPv4和IPv6地址类型各64条，域名地址类型为16条（暂不支持），总共80条。可配置策略条数为128条。

服务器主动发起的连接，如果命中非法外联策略，没有命中允许外联规则，则按照策略动作允许或者拒绝，拒绝会记录日志。

当用户访问网络服务时，可以在非法外联策略配置动作允许的外联规则，通过非法外联学习用户访问的所有服务器地址。学习完成后，将期望拒绝访问的服务器主机添加到非法外联策略配置的服务器对象中，用户就无法访问通过非法外联学习获取的网络服务器主机。

系统菜单点击“策略>其他防护>非法外联防护>非法外联学习”，进入非法外联学习页签。

在非法外联学习页面选择需要学习的服务器对象、学习时长等配置后，点击<应用>下发配置，点击开始学习中的<开始>按钮，开始学习，点击<停止>按钮，则停止学习。

注意：

学习过程中，学习配置项不能进行修改，停止学习后可进行修改，学习条数为500条，学满后自动停止学习。

系统菜单点击“策略>其他防护>非法外联防护>学习结果”，进入学习结果配置页签。可以通过该配置页签对学习结果删除、导出、查询和重置，也可以将学习到的规则添加到对应策略配置规则中。

非法外联的学习结果，包含服务器的地址、外联地址、端口和协议。点击<删除>按钮，可将学习到的结果从表中删除，点击<添加>按钮，则将学习到的规则添加到对应的策略的配置规则中。点击<导出>可将学习结果导出保存在本地。点击<查询>可根据协议类型、外联地址和端口进行查询。点击<重置>可重置查询结果。

提示：

重新开始学习后，原有的学习内容都会清空，学习结果是和服务地址关联的，只学习匹配到服务器地址的规则。
