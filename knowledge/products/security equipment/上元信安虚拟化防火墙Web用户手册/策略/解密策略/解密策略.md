# 解密策略

支持对IP和域名的主机进行审计。

随着市场网络安全的发展，更多的客户将原有的明文业务逐步迁移到SSL协议，能够很大程度保证客户访问网络的安全，数据不会轻易被第三方窥视。网络管理员也很难得到内网用户访问外网的相关数据，对企业内网有很大的风险。基于此问题，防火墙能够对SSL的流量进行解密，并完成明文流量的审计。

在系统菜单点击“策略>解密策略>SSL解密>SSL解密”，进入SSL解密配置页签。

在系统菜单点击“策略>解密策略>SSL解密>DNS解析”进入DNS解析页签。

在右侧“操作”列点击图标删除DNS域名解析结果。点击<新建>，可以新建DNS域名解析。

DNS解析配置项及详细说明如下

SSL解密需要在虚拟化防火墙上配置转发根证书，并将此根证书导入客户端的浏览器中，用来验证虚拟化防火墙签发的证书。如果不将根证书导入客户端的浏览器，会有告警提示。 SSL解密是基于模板的，所以需要配置SSL客户端和SSL服务端模板，模板的配置参数是建立SSL连接的证书、加密算法、SSL选项等。因此虚拟化防火墙支持SSL证书模板配置，用来分别与客户端和服务端建立SSL连接。

在系统菜单点击“策略>解密策略>SSL证书模板>SSL客户端”，进入SSL客户端配置页签，查看设备上SSL客户端配置。

在SSL客户端配置页签中，点击<新建>，可以添加SSL客户端。

SSL客户端配置项及详细说明如下：

在系统菜单点击“策略>解密策略>SSL证书模板>SSL服务端”，进入SSL服务端页签，查看设备上的SSL服务端配置。在右侧“操作”列点击图标删除添加的SSL服务端配置。

点击<新建>添加SSL服务端配置。
