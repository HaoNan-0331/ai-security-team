# 4.3 SMB 产品

## 4.3.1 ER 系列路由器掉线故障排查

ER 系列路由器采用全 Web 方式操作，定位于中小企业、网吧、经济酒店的宽带接入。根据客户网络的接入特点，所以定位故障的思路为：先检查硬件，其次检查配置，最后根据故障发生的网络层次进行定位（硬件→配置→二层→三层）。

### 检查端口指示灯状态

检查 ER 路由器端口指示灯，是否掉线时有：端口灯闪烁（端口有短时间中断），灯全亮/灯全灭，LAN 接口同时快闪烁的异常问题。

### 端口协商双工状态

检查 ER 路由器端口指示灯状态，如表 4-8 所示，是否存在协商异常问题。

**表 4-8 ER 路由器端口指示灯状态**

| 指示灯 | 状态 | 含义 |
|--------|------|------|
| Power | 绿灯常亮 | 供电正常 |
| Power | 灭 | 电源关闭或电源故障 |
| Link | 绿灯常亮 | 物理链路存在，并建立连接 |
| Link | 灭 | 物理链路存在，但未建立连接或物理链路不存在 |
| 100M | 绿灯常亮 | 端口工作在 100Mbps |
| 1000M | 绿灯常亮 | 端口工作在 1000Mbps |
| 10/100M | 灭 | 端口工作在 10/100Mbps 或链路未建立 |
| Link/Act | 绿灯常亮 | 链路建立 |
| Link/Act | 闪烁 | 端口在收发数据 |
| Link/Act | 灭 | 链路未建立 |

登录 ER 路由器 Web 页面，检查 WAN 端口配置是否正常。具体操作为页面向导：接口设置→WAN 设置→网口模式。

**检查 LAN 端口配置**是否正常。具体操作为页面向导：接口设置→LAN 设置→端口设置。

### 排查端口环路

当 LAN 接口 Link/Act 灯出现同时快闪烁时，需排查是否因端口环路造成掉线。

因 ER 路由器不支持 STP，建议先排查下联交换机是否存在环路。

ER 路由器可以通过设置"广播风暴抑制"为高，可暂时规避使得现象缓解，此时必须排查解决下联网络环路问题。具体操作为页面向导：接口设置→LAN 设置→端口设置。

### 收集抓包信息

通过 ER 路由器的镜像功能，收集端口灯异常时的抓包信息，反馈给呼叫中心分析。具体操作为页面向导：端口管理→端口配置→端口镜像。

### 配置是否正确

检查 ER 路由器是否按照用户手册中"典型组网配置举例"章节配置设备。《H3C ER 系列路由器用户手册》可在 H3C 中文网站中获取，具体操作为首页→服务支持→文档中心→路由器→H3C ER 系列路由器。

### 配置检查

检查 ER 路由器配置关键点是否按照要求配置，以下为具体配置内容：

1. **检查登录账号密码**是否有弱口令风险，建议关闭 Web 和 Telnet 的远程访问。具体操作为页面向导：设备管理→用户管理→远程管理。

2. **检查 ARP 防攻击**是否配置，执行"安全专区→ARP 安全→ARP 防护"命令，选中"检测到 ARP 攻击时，发送免费 ARP 报文"复选框。

免费 ARP 功能可防止 LAN 或 WAN 侧的主机受到 ARP 攻击。免费 ARP 发送间隔越小，主机抗 ARP 攻击能力越强，但对网络整体性能影响越大。当检测到 ARP 攻击时，立即发送免费 ARP 报文。

3. **检查 QoS 设置**。是否填写 WAN 真实带宽，起止 IP 是否包含全部上网 IP 地址，设置上下行限速是否合理。具体操作为页面向导：QoS 设置→流量管理→流量限制。

4. **检查 NAT 连接数限制**。具体操作为页面向导：QoS 设置→流量管理→流量限制→添加限速规则。

如果配置内容较多，应用内容复杂，建议先备份 ER 路由器设备配置，恢复出厂设置后逐步增加配置项测试。注意恢复出厂设置前，需备份用户设备配置。具体操作为页面向导：设备管理→基本管理→配置管理。

如果 ER 路由器 Web 页面内容与本指导有差异，建议升级 ER 路由器版本后进行配置。软件版本可从 H3C 中文网站获取，ER 路由器升级页面位置。具体操作为页面向导：设备管理→基本管理→软件升级。

说明：v1.x.x 系列软件版本，只能升级 v1.x 系列设备硬件；v2.x 系列软件，只能升级 v2.x 系列设备硬件。

### 二层是否掉线

根据用户反馈现象，是否因为二层原因造成的掉线，例如：
- 用户反馈掉线时，PC 之间互访也存在丢包或不通
- 根据掉线用户的分布，为单个交换机下所有用户类掉线或 PC 互访存在问题
- 复制内网文件时存在掉线问题

如出现上述现象，建议按网络层次排查是否为二层转发问题。

### 检查 ER 路由器 LAN 端口

检查 ER 路由器 LAN 接口是否存在大流量互访流量。具体操作为页面向导：系统监控→流量监控→流量。

如果 LAN 接口之间，短时间存在大流量情况，需要检查流量是否超过端口速率导致的丢包。

检查 ER 路由器 LAN 接口设置，关闭流控开关，同时在下联的交换机口关闭流控开关。

将用户组网修改，只接一个 LAN 接口上行，排除二层流量经过 ER 造成的掉线问题。依据此步骤可排除 ER 路由器二层掉线问题，需对下联交换机等设备进行故障排查。

### 三层掉线问题

根据上一步故障现象判断，用户流量经过 ER 路由器后产生掉线问题。

此类问题按用户可划分为部分用户掉线、全部用户掉线；按时间分类为特定时间、随机时间掉线；按特定条件划分为特定用户软件、特定流量触发掉线。

上述问题一般需要收集 ER 路由器信息，提供呼叫中心热线做进一步分析和观察。

三层掉线问题后续步骤中给出部分可确认故障类型作为判断和解决方法。如无法解决，需联系呼叫中心收集故障信息。

### 限速不当导致掉线

ER 路由器支持 IP QoS 限速，需根据实际带宽和使用情况进行限速，限速不当可能导致单个用户或部分用户异常掉线。

**例 1：** 单个用户掉线问题的排查。具体操作为页面向导：系统监控→流量监控→流量。

如果用户下行限速为 1024Kbps，反馈异常用户 IP 为 192.168.1.62，从统计看流量已经占满导致掉线。

此类问题需要用户关闭占用带宽的应用或在路由器上合理调整限速带宽。具体操作为页面向导：QoS 设置→流量管理→IP 流量限制，修改每 IP 上/下行流量上限。

**例 2：** 用户反馈部分用户存在掉线情况，需要查看总带宽是否已经占满。具体操作是：页面向导：系统监控→流量监控→端口流量。

如果用户总带宽为 2Mbps，但用户 WAN 接收速率已经接近 2Mbps，导致用户掉线。此类问题可在路由器上调整 WAN 总带宽或向运营商申请更高带宽。具体操作为页面向导：QoS 设置→流量管理→流量限制，调整 WAN 带宽。

### 设备异常导致掉线

出现所有用户掉线时，需登录 ER 路由器，检查 ER 路由器日志，是否有异常日志，例如：设备重启、WAN/LAN 接口 Up/Down 信息、CPU/内存占用高告警等日志。具体操作为页面向导：系统监控→系统日志→日志信息。

如果有出现掉线时间和 ER 路由器重启时间大致一致，需搭建本地日志服务器，将日志发送到内网服务器上，在服务器上搭建 Syslog Server 接收日志，判断是否为非法用户登录设备造成重启。

如有此情况可针对性地：修改 ER 路由器登录账号密码、关闭远程管理、关闭 SNMP 功能。

如果在掉线时，日志出现 WAN 或 LAN 物理接口 Up/Down 情况，可尝试更换网线。

如果在掉线时出现 ER 路由器 CPU、内存占用高，可尝试设备防攻击设置，开启异常流量防护功能或在掉线时抓包联系呼叫中心进行分析。

### 收集故障信息

当以上方法均无法解决时，需要收集 ER 路由器相关信息联系呼叫中心进行分析，相关操作如下：

1. **故障发生时的设备信息收集**，需登录 ER 路由器 Web 页面收集。具体操作为页面向导：系统监控→网络维护→一键导出。

2. **搭建日志服务器**，判断设备掉线异常时设备是否有相关日志上报。具体操作为页面向导：系统监控→系统日志→日志管理，设置日志级别为 debug(7)，选中发送到日志服务器复选框，填写内网 IP 地址。

3. **搭建抓包环境**，例如在 LAN3 口接一台抓包 PC，将 WAN 端口、LAN 业务端口镜像到 LAN3 口 PC 上。具体操作为页面向导：端口管理→端口配置→端口镜像。

将以上信息收集整理，并详细了解故障现象和网络拓扑后，求助呼叫中心处理。

## 4.3.2 ER 系列路由器忘记密码故障排查

ER 系列路由器一般采取 Web 方式登录和管理，其 Console 端口提供设备在 Web 无法登录或 Web 忘记账号密码时应急恢复的功能。

所以定位故障的思路为：先判断是否能进入 Web 登录页面，然后尝试是否能够通过"密码提示"找回密码，最后需通过 Console 口进行恢复。

### 是否可进入 Web 登录页面

输入 ER 路由器的 Web 登录地址，查看是否可以进入 Web 登录页面。

说明：
- Web 页面无法显示，请更换不同浏览器尝试登录
- ER 路由器 Web 登录地址可以通过以下方式获取：
  - 本地连接状态→详细信息→IPv4 默认网关
  - ER 路由器的默认的登录地址为 http://192.168.1.1，可尝试以此默认地址登录
  - 通过 DHCP 指纹获取，DHCP 获取的网关地址一般为 ER 路由器的 Web 登录地址

在 ER 路由器 LAN 接口下，如果管理 PC 通过 DHCP 无法获取地址，可尝试 Console 口下输入 `ip address` 命令并回车，即可显示路由器 LAN 口的 IP 地址，通过手工设置 PC 的默认网关地址登录。

```bash
<H3C>ip address
The LAN interface information:
IP address: 192.168.1.1
Mask: 255.255.255.0
```

### 输入密码错误时是否有密码提示

ER 路由器 Web 登录页面上输入默认的用户名和密码 admin/admin，尝试是否可以登录。用户密码错误，是否出现密码提示。

说明：
- 登录用户名为 admin，密码区分大小写，支持 1~31 位英文状态下的字符
- 如果页面没有"密码提示"信息，则可能的原因如下：
  1. ER 路由器软件版本非最新版本，此版本不具备"密码提示"功能
  2. 设置登录用户名密码时，未在密码管理内设置密码提示内容

### 通过 Console 口恢复

当 ER 路由器无法登录到 Web 页面，可通过 Console 方式登录 ER 路由器进行命令行方式恢复。

通过串口登录命令行界面，通过 password 命令来设置新密码。

**命令：** `password`

```bash
<H3C>password
Changing password for admin
Enter the new password (minimum 1 maximum of 31 English characters):
Enter new password:
Reenter new password:
Password has been set successfully
<H3C>
```

如果 password 命令需要输入原密码，则只能输入 `restore default` 命令恢复到出厂设置。

说明：恢复出厂设置后，路由器的用户名、密码以及 IP 地址等所有设置都会被恢复到默认设置。

**命令：** `restore default`

```bash
<H3C>password
Changing password for admin
Old password:
Incorrect Password for admin
<H3C>restore default
This command will restore the default configuration and reboot the system
The current configuration will be lost if you continue.
Continue? [Y/N]:y
<H3C>
```

管理 PC 串口连接 ER 路由器方法如下：

通过配置线缆（DB 9 转成 RJ45 的通用转换线）连接到计算机。

配置管理计算机参数，以 Windows XP 附带的超级终端为例，打开管理计算机，在管理计算机 Windows 界面上单击"开始"→"所有程序"→"附件"→"通讯"命令，运行终端仿真程序。在"名称"文本框中键入新建连接的名称。

在"连接时使用"下拉框中选择进行连接的串口（请确保选择的串口应与配置线缆实际连接的串口相一致），单击"确定"按钮。

在串口的属性对话框中设置相关参数，需注意，每秒位数（波特率）：9600，数据流控制：无。单击"确定"按钮。

单击"文件"→"属性"→"设置"命令，进入属性设置窗口。选择"终端仿真类型"为"自动检测"，单击"确定"按钮。

按 Enter 键后，即可登录路由器，且终端上显示命令行提示符 `<H3C>`。

### 通过密码提示找回

通过 ER 路由器的密码提示功能，根据密码提示信息找回登录密码。

### 通过页面中忘记密码恢复

通过单击 ER 路由器的登录页面"忘记密码?"链接，通过链接内容"忘记密码怎么办"，通过串口恢复到出厂设置。

说明：恢复出厂设置后，路由器的用户名、密码以及 IP 地址等所有设置都会被恢复到默认设置。

具体方法如下：
1. 连接配置计算机到路由器
2. 将管理计算机串口通过配置线缆与路由器的 Console 连接
3. 配置超级终端参数
4. 在串口下输入命令 `restore default`
