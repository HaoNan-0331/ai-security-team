# 3.6 AAA

## 3.6.1 RADIUS 故障排查

RADIUS 协议承载于 UDP 协议，是基于 C/S 架构的，客户端和服务器都要排查。所以定位故障的思路是分别检查作为 RADIUS 客户端的设备侧和 RADIUS 服务器。

### 查看用户是否在线

在设备上查看用户是否在线。

**命令：** `display connection`
**命令：** `display connection ucibindex ucibindex-number`

例如：通过命令查看在线用户信息，可以确认用户名 admin 的用户已经在线，其用户索引为 134，后面跟 ucibindex 参数及对应用户的索引 134 可查看对应在线用户 admin 的详细信息。

```bash
<H3C>display connection
Slot: Index = 134
Username: admin@system
IP: 10.153.47.44
IPv6: N/A

<H3C>display connection ucibindex 134
Slot: Index = 134
Username: admin@system
IP: 10.153.47.44
IPv6: N/A
Access: Admin
AuthMethod: PAP
Port Type: Virtual
Port Name: N/A
Initial VLAN: N/A
Authorization VLAN: N/A
ACL Group: Disable
User Profile: N/A
CAR: Disable
Priority: Disable
SessionTimeout: N/A
Terminate Action: N/A
Start: 2013-05-06 14:08:43
Current: 2013-05-06 14:50:47
Online: 0h 42m 04s
Total connection matched
```

### 检查用户名密码

确保客户端输入的用户名密码正确。

### 检查设备与 RADIUS Server 是否互通

确保设备到 RADIUS Server IP 地址和端口可达。

### 检查 Key 与 RADIUS Server 是否一致

查看设备侧配置的 RADIUS Key 和 RADIUS Server 侧配置的 Key 是否一致。

**命令：** `display this`

例如：通过命令查看 Key 配置。

```bash
[H3C]radius scheme imc
[H3C-radius-imc]display this
radius scheme imc
primary authentication 1.1.1.1
primary accounting 1.1.1.1
key authentication h3c
key accounting h3c
user-name-format without-domain
```

### 检查 Domain 或 RADIUS Scheme 配置是否正确

如果设备要做 EAD 或者下发 H3C 私有 RADIUS 属性，则需要将服务类型配置为 extended。

**命令：** `display this`

例如：RADIUS Scheme 视图下查看服务类型是否为 extended。

```bash
[H3C-radius-imc]display this
server-type extended
primary authentication 1.1.1.1
primary accounting 1.1.1.1
secondary authentication 1.1.1.17
secondary accounting 1.1.1.17
```

我们设备默认的 domain 域是 system，如果不指定域后缀，用户认证的时候都会到默认的 system 域去认证，要检查默认域是否配置为用户的认证域。另外不管是否存在计费和授权，在 domain 下都要同时指定认证、授权、计费的 radius-scheme，三者同配，缺一不可。

**命令：** `display this`
**命令：** `domain default enable h3c`

例如：查看配置中认证、授权、计费的引用，并配置用户认证域 H3C 为默认的 domain。

```bash
[H3C]domain h3c
[H3C-isp-h3c]display this
domain h3c
authentication login radius-scheme imc
authorization login radius-scheme imc
accounting login radius-scheme imc
access-limit disable
state active
idle-cut disable
self-service-url disable
[H3C]domain default enable h3c
```

### 查看 RADIUS Scheme 状态信息

不仅仅要确认到 RADIUS 服务器路由可达，还要确认对应认证、计费、授权服务器是否端口可达。

**命令：** `display radius scheme`

例如：通过命令查看 radius scheme 状态信息，确认其状态为 Active。

```bash
<H3C>display radius scheme imc
SchemeName: imc
Index: 1
Type: extended
Primary Auth Server:
IP: 1.1.1.1
Port: 1812
State: active
Encryption Key: N/A
VPN instance: N/A
Probe username: N/A
Probe interval: N/A
Primary Acct Server:
IP: 1.1.1.1
Port: 1813
State: active
Encryption Key: N/A
VPN instance: N/A
```

### NAS-IP 配置是否正确

检查服务器侧是否配置了 NAS-IP 以及配置的 NAS-IP 是否与设备指定的 NAS-IP 一致。设备的 NAS-IP 为 10.1.1.200，则在 iMC 侧配置的接入设备 IP 也要是 10.1.1.200。

### RADIUS 进程是否正常启动

查看服务器侧 RADIUS 进程是否正常启动，是否存在应用端口号被占用的情况，以及是否配置 RADIUS 代理。

### 是否下发了设备不支持的属性

服务器可以下发各种 RADIUS 属性，有些属性对格式有要求，需注意下发的属性格式在设备上是否能够支持。如果下发了设备不支持的属性会导致认证失败。

常用的 RADIUS 属性有：用户权限、ACL、VLAN、CAR 限速。常见的 RADIUS 属性如下：

| 属性 | 属性编号 | 格式 | 具体定义 |
|------|----------|------|----------|
| Input-Peak-Rate | 63 | Integer | 用户接入到 NAS 的峰值速率，以 bps 为单位 |
| Input-Average-Rate | 64 | Integer | 用户接入到 NAS 的平均速率，以 bps 为单位 |
| Input-Basic-Rate | 65 | Integer | 用户接入到 NAS 的基本速率，以 bps 为单位 |
| Output-Peak-Rate | 66 | Integer | 从 NAS 到用户的峰值速率，以 bps 为单位 |
| Output-Average-Rate | 67 | Integer | 从 NAS 到用户的平均速率，以 bps 为单位 |
| Output-Basic-Rate | 68 | Integer | 从 NAS 到用户的基本速率，以 bps 为单位 |
| Exec-Privilege | 29 | Integer | 用户优先级，对于 EXEC 用户，用户的优先级 |
| Tunnel-Private-Group-ID | 81 | Integer | VPDN 组号，取值为 [1-4094] |
| Tunnel-Assignment-ID | 82 | Integer | 参照 RFC 定义 |
| Filter-ID | 11 | Text | 访问控制列表的名字 |

### 查看认证失败或下线原因

RADIUS 服务器都会有对应的日志记录用户认证失败或下线的原因。常见的下线原因可以参考下面的说明。

**User-Request**：这种情况属于用户请求下线，一般为客户端主动下线，需要检查客户端。

**Lost Carrier**：这种情况一般为客户端与设备间握手报文丢失导致，因为 H3C 设备的握手功能是私有的，在跟第三方客户端配合时，需要关闭握手。

关闭握手的命令为：`undo dot1x handshake`

例如：关闭端口 GE1/0/10 下的 dot1x 握手功能。

```bash
[H3C]int GigabitEthernet1/0/10
[H3C-GigabitEthernet1/0/10]undo dot1x handshake
```

**Idle Timeout**：这种情况为闲置超时下线。

**Session Timeout**：这种情况为会话超时，一般为计费不足或者进行了时间段限制。

## 3.6.2 TACACS 故障排查

TACACS 协议是基于 C/S 架构承载于 TCP 之上，客户端和服务器都要排查。所以定位故障的思路是分别检查作为 TACACS 客户端的设备侧和 TACACS 服务器。

### 查看用户是否在线

在设备上查看用户是否在线。

**命令：** `display connection`
**命令：** `display connection ucibindex ucibindex-number`

例如：通过命令查看 admin 用户已经在线，其用户索引为 134，后面跟 ucibindex 参数及对应用户的索引 134 可查看对应在线用户 admin 的详细信息。

```bash
<H3C>display connection
Slot: Index = 134
Username: admin@system
IP: 10.153.47.44
IPv6: N/A

<H3C>display connection ucibindex 134
Slot: Index = 134
Username: admin@system
IP: 10.153.47.44
IPv6: N/A
Access: Admin
AuthMethod: PAP
Port Type: Virtual
Port Name: N/A
Initial VLAN: N/A
Authorization VLAN: N/A
ACL Group: Disable
User Profile: N/A
CAR: Disable
Priority: Disable
SessionTimeout: N/A
Terminate Action: N/A
Start: 2013-05-06 14:08:43
Current: 2013-05-06 14:50:47
Online: 0h 42m 04s
Total connection matched
```

### 检查用户名密码

检查输入的用户名密码是否正确。

### 检查设备与 TACACS Server 是否互通

确保设备与 TACACS Server 之间 IP 和端口可达。

### 检查 Key 与 TACACS Server 是否一致

查看设备侧配置的 TACACS Key 和 TACACS Server 侧配置的 Key 是否一致。

**命令：** `display this`

例如：通过命令查看 Key 配置，这里配置的 Key 是 h3c，在 Server 侧也要配置一样的 Key。

```bash
[H3C]hwtacacs scheme Tacacs
[H3C-hwtacacs-Tacacs]display this
hwtacacs scheme Tacacs
primary authentication 1.1.1.1
primary authorization 1.1.1.1
primary accounting 1.1.1.1
key authentication h3c
key authorization h3c
key accounting h3c
user-name-format without-domain
```

### 检查 Domain 域配置是否正确

H3C 设备默认的 domain 域是 system，如果不指定域后缀，用户认证的时候都会到默认的 system 域去认证，要检查默认域是否配置为用户的认证域。另外不管是否存在计费和授权，在 domain 下都要同时指定认证、授权、计费的 hwtacacs-scheme，否则默认都走本地容易导致认证失败。

**命令：** `domain default enable domain-name`

例如：查看配置中认证、授权、计费引用的 TACACS Scheme 是 Tacacs，并配置用户认证域 H3C 为默认的 domain。

```bash
[H3C]domain H3C
[H3C-isp-H3C]display this
domain H3C
authentication login hwtacacs-scheme Tacacs
authorization login hwtacacs-scheme Tacacs
accounting login hwtacacs-scheme Tacacs
access-limit disable
state active
idle-cut disable
self-service-url disable
[H3C]domain default enable H3C
```

### 检查是否配置命令行记账与授权

如果要对用户进行命令行授权和记账则需要在 user-interface vty 接口下添加相应配置。

**命令：** `command authorization`
**命令：** `command accounting`

例如：对 user-interface vty 0~4 配置命令行授权与记账功能。

```bash
[H3C]user-interface vty 0 4
[H3C-line-vty0-4]authentication mode scheme
[H3C-line-vty0-4]command authorization
[H3C-line-vty0-4]command accounting
[H3C-line-vty0-4]idle-timeout 120
[H3C-line-vty0-4]screen-length 30
```

### 检查 NAS-IP 配置是否正确

NAS-IP 是设备和服务器进行通信的 IP 地址，服务器会对 NAS-IP 进行检查确定是否是合法的 NAS 设备。要检查 TACACS 服务器侧是否配置了 NAS-IP 以及配置的 NAS-IP 是否与设备侧指定的 NAS-IP 一致。

### 检查 TACACS 进程是否正常启动

查看服务器侧 TACACS 进程是否正常启动，是否存在应用端口号被占用的情况。

TACACS 默认使用的认证、授权、计费端口号都是 TCP 49。

### 检查是否配置命令行授权

检查命令行授权配置格式是否正确。

### 检查用户是否有远程认证的权限

确认在服务器上已经建立了相应的认证用户，并且赋予了用户远程认证权限。
