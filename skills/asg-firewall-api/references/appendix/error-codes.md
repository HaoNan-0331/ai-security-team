# 错误码参考

## 概述

ASG防火墙API在操作失败时会返回错误码和错误描述。本文档列出了常见的错误码及其含义。

## 错误响应格式

```json
{
    "code": "错误码",
    "str": "错误描述信息"
}
```

## 通用错误码

### 认证和权限相关

| 错误码 | 错误描述 | 说明 | 解决方案 |
|--------|----------|------|----------|
| -1 | 认证失败 | Token无效或已过期 | 检查Token是否正确，重新生成Token |
| -2 | 权限不足 | 当前Token无权限执行该操作 | 联系管理员分配相应权限 |

### 参数错误

| 错误码 | 错误描述 | 说明 | 解决方案 |
|--------|----------|------|----------|
| 250 | 参数错误 | 请求参数格式或值不正确 | 检查请求参数格式和取值范围 |
| 251 | 必填参数缺失 | 缺少必填的请求参数 | 补充必填参数 |
| 252 | 参数值超出范围 | 参数值不在允许的范围内 | 调整参数值到有效范围 |

### 对象相关错误

| 错误码 | 错误描述 | 说明 | 解决方案 |
|--------|----------|------|----------|
| 87 | 目标策略不存在 | 指定ID的策略不存在 | 检查策略ID是否正确 |
| 106 | 名称冲突 | 对象名称已存在 | 使用不同的名称或先删除现有对象 |
| 107 | 对象名称不存在 | 指定名称的对象不存在 | 检查对象名称是否正确 |
| 108 | 对象被引用 | 对象正在被其他配置引用 | 先解除引用关系再删除 |
| 109 | 预定义对象不能删除 | 尝试删除系统预定义对象 | 预定义对象不可删除 |
| 110 | 对象数量超限 | 已达到对象数量上限 | 删除不用的对象或升级授权 |

### 接口和地址相关

| 错误码 | 错误描述 | 说明 | 解决方案 |
|--------|----------|------|----------|
| 22 | 非法的IP地址 | IP地址格式不正确 | 检查IP地址格式 |
| 50 | IP地址错误 | IP地址不符合要求 | 使用有效的IP地址 |
| 90 | 接口不存在 | 指定的接口不存在 | 检查接口名称是否正确 |
| 116 | 非法的MAC地址 | MAC地址格式不正确 | 检查MAC地址格式（xx:xx:xx:xx:xx:xx） |
| 193 | IP地址格式错误 | IPv6地址格式不正确 | 检查IPv6地址格式 |

### 策略相关错误

| 错误码 | 错误描述 | 说明 | 解决方案 |
|--------|----------|------|----------|
| 122 | 该策略已经存在 | 尝试添加已存在的策略 | 检查策略是否已存在 |
| 123 | 策略数量超限 | 已达到策略数量上限 | 删除不用的策略或升级授权 |
| 124 | 策略ID冲突 | 策略ID已被使用 | 使用不同的策略ID |
| 125 | 策略ID无效 | 策略ID不在有效范围内 | 使用1-10000范围内的ID |

### 文件和上传相关

| 错误码 | 错误描述 | 说明 | 解决方案 |
|--------|----------|------|----------|
| -137 | 无效的系统升级文件 | 升级文件格式或内容不正确 | 使用正确的升级文件 |
| -221 | 非法的特征库文件 | 特征库文件格式不正确 | 使用正确的特征库文件 |

### 系统相关错误

|--------|----------|------|----------|
| 248 | 没有配置DNS服务器 | 需要DNS服务器但未配置 | 先配置DNS服务器 |
| 100 | 系统忙 | 系统正在处理其他任务 | 稍后重试 |

### 操作相关错误

## 错误处理建议

### 1. 检查错误码

```python
response = requests.post(url, json=data)
if response.status_code == 200:
    result = response.json()
    if 'code' in result:
        # 错误响应
        print(f"错误: {result['code']} - {result['str']}")
    else:
        # 成功响应
        print("操作成功")
```

### 2. 常见错误处理流程

```
1. 解析响应
   ↓
2. 检查是否有code字段
   ↓
3. 根据错误码分类处理
   - 认证错误 → 重新获取Token
   - 参数错误 → 检查并修正参数
   - 对象不存在 → 先创建对象
   - 名称冲突 → 使用不同名称
   ↓
4. 记录错误日志
   ↓
5. 根据需要决定是否重试
```

### 3. 重试策略

某些错误适合重试：
- 系统忙 (code: 100)
- 网络超时

不适合重试的错误：
- 参数错误 (code: 250)
- 对象不存在 (code: 87, 107)
- 权限不足 (code: -2)

### 4. 错误日志记录

建议记录以下信息：
- 错误码和错误描述
- 请求URL和方法
- 请求参数
- 时间戳
- 相关的业务上下文

```python
{
    "timestamp": "2024-01-28 12:00:00",
    "error_code": "106",
    "error_msg": "名称冲突",
    "endpoint": "/api/address",
    "method": "POST",
    "params": {"name": "test_addr"},
    "context": "批量添加地址对象"
}
```
